## Build Errors

### ❌ Error: CS1061 - Entity Property Not Found (CRITICAL)

**Date Encountered:** 2025-11-27

**Error Message:**
```
error CS1061: 'Role' bir 'RoleID' tanımı içermiyor ve 'Role' türünde bir ilk bağımsız değişken kabul eden hiçbir erişilebilir 'RoleID' genişletme yöntemi bulunamadı
error CS1061: 'Permission' bir 'PermissionID' tanımı içermiyor...
error CS1061: 'User' bir 'UserID' tanımı içermiyor...
```

**Root Cause:**
Duplicate entity files in `IntranetPortal.Infrastructure` folder were **overriding** the correct entities from `IntranetPortal.Domain.Entities`. These files were:
- Auto-generated by **EF Core Scaffold** from database
- Using incorrect property names (e.g., `UserId` instead of `UserID`)
- Located directly in Infrastructure root folder

**Files Involved:**
```
IntranetPortal.Infrastructure/
├── User.cs              ❌ WRONG - Scaffolded entity
├── Role.cs              ❌ WRONG - Scaffolded entity
├── Permission.cs        ❌ WRONG - Scaffolded entity
├── Birim.cs             ❌ WRONG - Scaffolded entity
├── RolePermission.cs    ❌ WRONG - Scaffolded entity
├── UserBirimRole.cs     ❌ WRONG - Scaffolded entity
├── AuditLog.cs          ❌ WRONG - Scaffolded entity
├── SystemSetting.cs     ❌ WRONG - Scaffolded entity
├── UploadedFile.cs      ❌ WRONG - Scaffolded entity
├── IntranetDbContext.cs ❌ WRONG - Scaffolded DbContext
└── Class1.cs            ❌ Template file
```

**Solution:**
Delete ALL auto-generated entity files from Infrastructure layer:

```bash
cd IntranetPortal.Infrastructure
rm -f User.cs Role.cs Permission.cs Birim.cs RolePermission.cs
rm -f UserBirimRole.cs AuditLog.cs SystemSetting.cs UploadedFile.cs
rm -f IntranetDbContext.cs Class1.cs
```

Then clean and rebuild:
```bash
dotnet clean
dotnet build
```

**Why This Happened:**
1. Developer ran `dotnet ef dbcontext scaffold` command in Infrastructure project
2. EF Core generated entities directly in project root
3. These scaffolded entities conflicted with manually-created Domain entities
4. C# compiler prioritized local files over referenced project files

**Verification:**
After deletion, build should succeed:
```
Oluşturma başarılı oldu.
    0 Uyarı
    0 Hata
```

**Affected Files Count:** 74 compilation errors across Configuration and Seeding files

## Preventive Measures

### 🛡️ Build Failure Prevention Checklist

#### Before Running `dotnet build`:

1. **Check for Duplicate Entities:**
   ```bash
   cd IntranetPortal.Infrastructure
   ls *.cs  # Should only show files in Data/, Configurations/, Repositories/
   ```

2. **Verify Entity Locations:**
   - ✅ Entities should ONLY exist in: `IntranetPortal.Domain/Entities/`
   - ❌ Entities should NEVER exist in: `IntranetPortal.Infrastructure/` (root)

3. **Clean Before Build:**
   ```bash
   dotnet clean
   dotnet build
   ```

#### Project Structure Validation:

**CORRECT Structure:**
```
IntranetPortal.Infrastructure/
├── Data/
│   ├── ApplicationDbContext.cs    ✅ Correct DbContext
│   └── Seeding/
│       └── DatabaseSeeder.cs
├── Configurations/
│   ├── UserConfiguration.cs
│   ├── RoleConfiguration.cs
│   └── ...
├── Repositories/
│   └── (repository implementations)
└── IntranetPortal.Infrastructure.csproj
```

**INCORRECT Structure (DO NOT CREATE):**
```
IntranetPortal.Infrastructure/
├── User.cs                 ❌ DELETE
├── Role.cs                 ❌ DELETE
├── IntranetDbContext.cs    ❌ DELETE
└── ...
```

#### Entity Framework Commands - Safe Usage:

**✅ SAFE Commands:**
```bash
# Add migration (always run from solution root)
dotnet ef migrations add MigrationName --project IntranetPortal.Infrastructure

# Update database
dotnet ef database update --project IntranetPortal.Infrastructure

# Generate SQL script
dotnet ef migrations script -o migration.sql --project IntranetPortal.Infrastructure
```

**❌ DANGEROUS Commands (NEVER USE):**
```bash
# ❌ DO NOT scaffold database to code
dotnet ef dbcontext scaffold "..." Npgsql.EntityFrameworkCore.PostgreSQL

# ❌ DO NOT scaffold with output to Infrastructure
dotnet ef dbcontext scaffold "..." Npgsql.EntityFrameworkCore.PostgreSQL -o .
```

## Recovery Procedures

### 🚨 Emergency Build Fix Procedure

If you encounter CS1061 errors about missing properties:

1. **Stop all builds immediately**
2. **Search for duplicate files:**
   ```bash
   cd backend
   find . -name "User.cs" -o -name "Role.cs"
   ```
3. **Delete any entity files outside Domain/Entities/**
4. **Clean solution:**
   ```bash
   dotnet clean
   rm -rf */obj */bin
   ```
5. **Rebuild:**
   ```bash
   dotnet restore
   dotnet build
   ```

### 🔍 Diagnostic Commands

```bash
# Check what files are being compiled
dotnet build /v:detailed > build.log 2>&1

# Search for specific error patterns
grep "CS1061" build.log

# Verify Domain.dll was built correctly
dotnet build IntranetPortal.Domain
# Should succeed with 0 errors

# Check if Infrastructure references Domain correctly
grep -A5 "ProjectReference" IntranetPortal.Infrastructure/*.csproj
# Should show: <ProjectReference Include="..\IntranetPortal.Domain\IntranetPortal.Domain.csproj" />
```

## Best Practices

### ✅ DO:
- Keep entities ONLY in `IntranetPortal.Domain/Entities/`
- Use Code-First approach with EF Core migrations
- Run `dotnet clean` before major builds
- Verify project structure after pulling changes
- Document any new errors in this file

### ❌ DON'T:
- Never scaffold database to code in Infrastructure project
- Never create entity files outside Domain layer
- Never ignore CS1061 errors (they indicate duplicate entities)
- Never commit scaffold-generated files to version control
- Never run EF commands without `--project` parameter
