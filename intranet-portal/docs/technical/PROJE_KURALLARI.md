# Proje Geliştirme Kuralları ve Standartları

Bu proje için **Antigravity Rules** havuzundan seçilen ve teknoloji yığınımıza (C#, .NET 9, React 19, TS, PostgreSQL) uygun olan kurallar aşağıdadır.

## 1. Backend & Veritabanı
*   **ASP.NET Core C# Expert:** .NET 9 ve Clean Architecture prensipleri.
*   **PostgreSQL Expert:** Verimli şema tasarımı, indexleme ve JSONB kullanımı.
*   **Database Design & Normalization:** Veri bütünlüğü ve normalizasyon standartları.
*   **SQL Query Optimization:** Performanslı sorgular ve N+1 problemlerinin önlenmesi.
*   **Database Migration Strategies:** Güvenli şema güncellemeleri ve versiyonlama.
*   **Redis & Caching Strategies:** Etkili önbellekleme stratejileri (In-Memory veya Redis).
*   **REST API Design Patterns Expert:** Tutarlı, kaynak odaklı API tasarımı.
*   **Security & Penetration Testing:** OWASP standartlarına uygun güvenlik kontrolleri.
*   **Web Security Best Practices Expert:** [YENİ] HTTPS, XSS ve CSP gibi modern web güvenliği standartları.
*   **API Testing (Postman, REST Assured):** [YENİ] API kontratlarının ve iş mantığının doğrulanması.

## 2. Frontend (React & TypeScript)
*   **React and modern JavaScript:** Modern React 19 ve ES6+ standartları.
*   **React Hooks Best Practices:** Hook kurallarına sıkı uyum ve custom hook kullanımı.
*   **React Performance Optimization:** Gereksiz render'ları önleme ve performans ölçümü.
*   **Advanced React Component Patterns:** Temiz ve yönetilebilir bileşen mimarisi.
*   **Tailwind CSS Expert:** Utility-first CSS yaklaşımı.
*   **Advanced TypeScript Patterns:** Tip güvenliği (Type Safety) ve jenerik yapılar.
*   **TypeScript Tooling & Ecosystem:** Doğru araç kullanımı.
*   **Semantic HTML & Accessibility Expert:** [YENİ] Erişilebilir ve semantik HTML yapısı (WAI-ARIA).
*   **TypeScript Strict Mode & Safety:** [YENİ] Strict mode ile tip hatalarının erken tespiti.

## 3. Genel Kalite & Test
*   **Unit Testing Best Practices:** İzole, hızlı ve güvenilir birim testler.
*   **Integration Testing Strategies:** Modüller arası uyumun doğrulanması.
*   **Performance Testing (k6, JMeter):** Yük altındaki performansın ölçülmesi.
*   **Test-Driven Development (TDD):** [YENİ] Test odaklı geliştirme yaklaşımı (Red-Green-Refactor).
